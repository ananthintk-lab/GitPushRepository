## Codebase Patterns
- Accept filter params as `String` in Spring MVC controllers and convert to enum manually to safely handle empty string from "All" option in HTML select forms.
- Spring Boot 4.x removed Flyway auto-configuration from spring-boot-autoconfigure. Use `schema.sql` + `spring.sql.init.mode=always` for DB initialization instead.
- Spring Boot 4.x uses separate module JARs (spring-boot-jpa, spring-boot-jdbc) for JPA/JDBC auto-config — not in spring-boot-autoconfigure.
- `JpaDependsOnDatabaseInitializationDetector` in spring-boot-jpa ensures JPA waits for schema.sql to run before validating.
- The original pom.xml had invalid `spring-boot-starter-data-jpa-test` dependency — replaced with `spring-boot-starter-test`.
- Use `spring.jpa.hibernate.ddl-auto=none` when relying on schema.sql for table creation.
- H2 in-memory URL: `jdbc:h2:mem:githubdemo;DB_CLOSE_DELAY=-1`
- UI layer uses Thymeleaf (`spring-boot-starter-thymeleaf`); templates go in `src/main/resources/templates/`.
- Use `ApplicationRunner` + `DataInitializer` for seeding sample data on startup (guard with `repository.count() == 0`).
- Thymeleaf priority badge pattern: `th:classappend="${task.priority.name()}"` appends the enum name as CSS class; define `.HIGH`, `.MEDIUM`, `.LOW` CSS rules for color-coding.
