## Codebase Patterns
- Spring Boot 4.x removed Flyway auto-configuration from spring-boot-autoconfigure. Use `schema.sql` + `spring.sql.init.mode=always` for DB initialization instead.
- Spring Boot 4.x uses separate module JARs (spring-boot-jpa, spring-boot-jdbc) for JPA/JDBC auto-config — not in spring-boot-autoconfigure.
- `JpaDependsOnDatabaseInitializationDetector` in spring-boot-jpa ensures JPA waits for schema.sql to run before validating.
